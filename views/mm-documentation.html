UI Design

The design is was intended to duplicate the responsiveness of an app; and have no direct HTTP get requests to load entire pages; i.e. it’s designed as a single-page web application.

Color:

The colors chosen were intended to contrast well with content, which is presumably much less likely to be as dark (as most photos are taken during the day); dark colors hopefully increase the contrast between the content and the UI that surrounds it. Four main colors were chosen: teal, dark-blue, black and white; 1-2 variations of each colors are used.

Texture:

Decided to not use texture except on the splash screen; a white background would probably be less distracting. The texture of the world map on the splash screen is more thematically appropriate.

Emphasis and Hierarchy:

The fixed top & left navigation bars of the page are meant to serve as constant hierarchy indicators to the user.

Contrast:

The navigational elements were colored dark so as to contrast with the content and distinguish the content from the UI chrome more easily.

Repetition:

All thumbnails use the same width and height dimensions so as to keep the repeating of thumbnails consistent.

Rhythm:

The thumbnail borders, and button borders maintain a consistent border radius size. The animated elements on the splash screen also use similar easing functions.

Harmony:

Only quadrilateral shapes were used to maintain a sense of harmony.

Unity:

Tried to keep the design elements relating to one another, no elements that stood out to much from others.

Proximity:

The thumbnails are kept closer together than they are further apart from the user so as to allow the user to intuit they are part of the same group.

Alignment:

Used Bootstrap’s 12-column grid system to keep alignment naturally responsive to the width of the viewport.

Typography:

Text is truncated with an ellipsis so as to indicate that content overflows.
